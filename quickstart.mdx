---
title: Quickstart
description: Run a complete Structure-from-Motion reconstruction on a sample dataset in minutes.
---

This guide walks through a full end-to-end reconstruction workflow using a publicly available dataset. By the end you will have a sparse 3D model you can open in the COLMAP GUI.

<Note>
GLOMAP requires a COLMAP database as input. If you already have a database, skip to [Run GLOMAP](#run-glomap).
</Note>

<Steps>
  <Step title="Download a sample dataset">
    Download one of the sample datasets from [demuc.de/colmap/datasets](https://demuc.de/colmap/datasets) and place it under a `data/` folder. The examples below use the `south-building` dataset:

    ```
    data/
    └── south-building/
        └── images/
    ```
  </Step>

  <Step title="Extract features">
    Use `colmap feature_extractor` to detect and describe keypoints in each image and write the results to a database:

    ```shell
    colmap feature_extractor \
        --image_path    ./data/south-building/images \
        --database_path ./data/south-building/database.db
    ```
  </Step>

  <Step title="Match features">
    Match keypoints across image pairs. For small datasets, exhaustive matching is straightforward:

    ```shell
    colmap exhaustive_matcher \
        --database_path ./data/south-building/database.db
    ```

    <Tip>
    For larger datasets, use `sequential_matcher` or `vocab_tree_matcher` instead to keep runtime manageable:

    ```shell
    colmap sequential_matcher --database_path ./data/south-building/database.db
    ```
    </Tip>
  </Step>

  <Step title="Run GLOMAP">
    With features and matches in the database, run the global mapper to produce a sparse reconstruction:

    ```shell
    glomap mapper \
        --database_path ./data/south-building/database.db \
        --image_path    ./data/south-building/images \
        --output_path   ./output/south-building/sparse
    ```

    GLOMAP writes the reconstruction under `./output/south-building/sparse/` in the standard COLMAP sparse reconstruction format.

    For inline help on all available flags, run:

    ```shell
    glomap mapper -h
    ```
  </Step>

  <Step title="Visualize the results">
    Open the reconstruction in the COLMAP GUI:

    ```shell
    colmap gui --import_path ./output/south-building/sparse/0
    ```

    Alternatively, [rerun.io](https://rerun.io/examples/3d-reconstruction/glomap) supports visualizing COLMAP and GLOMAP outputs directly.
  </Step>
</Steps>

## Starting from an existing database

If you already have a COLMAP database (for example, the `gerrard-hall` dataset), you can skip feature extraction and matching and run GLOMAP directly:

```shell
glomap mapper \
    --database_path ./data/gerrard-hall/database.db \
    --image_path    ./data/gerrard-hall/images \
    --output_path   ./output/gerrard-hall/sparse
```

## Output format

GLOMAP writes reconstructions in the [COLMAP sparse reconstruction format](https://colmap.github.io/format.html#sparse-reconstruction). The output directory contains:

| File | Contents |
|------|----------|
| `cameras.bin` / `cameras.txt` | Camera intrinsics for each camera model |
| `images.bin` / `images.txt` | Pose and feature observations for each image |
| `points3D.bin` / `points3D.txt` | 3D point cloud with track information |

You can inspect the reconstruction programmatically using `pycolmap` in Python or by linking against COLMAP's C++ library interface.

## Available commands

<CardGroup cols={3}>
  <Card title="mapper" icon="map">
    Full global SfM pipeline from a COLMAP database.

    ```shell
    glomap mapper \
      --database_path DB \
      --image_path IMAGES \
      --output_path OUTPUT
    ```
  </Card>
  <Card title="mapper_resume" icon="rotate-ccw">
    Resume a previous reconstruction from an existing model.

    ```shell
    glomap mapper_resume \
      --input_path MODEL_INPUT \
      --output_path MODEL_OUTPUT
    ```
  </Card>
  <Card title="rotation_averager" icon="refresh-cw">
    Run the standalone rotation averaging module.

    See the [rotation averager guide](/commands/rotation-averager) for full usage details.
  </Card>
</CardGroup>
